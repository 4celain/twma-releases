# 개인정보 처리 요약

[English version](PRIVACY_EN.md)

TWMA는 창 배치 복원에 필요한 정보를 이 Mac 안에만 저장합니다.

## 로컬에 저장하는 정보

- 작업공간 이름과 마지막 변경 시각
- 저장 대상 앱의 이름과 bundle identifier
- 새 작업공간을 만들거나 업데이트할 때 활성 데스크탑에 보이는 일반 창의 제목, 창 종류와 순서 및 마지막 전면 앱·창
- 모니터 식별자·이름·논리 해상도·배치·배율·회전·내장 여부
- 모니터별 레이아웃과 창별 배치 영역 번호
- 복원할 때 앱을 실행할지 여부와 자동 복원 대상으로 선택된 작업공간
- 사용자가 직접 지정한 작업공간 복원 단축키와 모든 모니터에 공통인 메뉴 고정 레이아웃 1~4번 슬롯 단축키 조합
- 사용자가 지정한 앱 배치 방식과 앱 이름·bundle identifier
- 언어, 드래그 배치, 미리보기 스타일, Dock 공간 확보, 창 사이 간격, 자동 복원과 로그인 실행 설정

작업공간 데이터는 `~/Library/Application Support/TWMA/profiles-v1.json`에 저장합니다. `새 작업공간`을 만들 때 현재 창 목록과 배치를 새로 저장하고, 저장된 작업공간에서 `업데이트`를 실행하면 해당 작업공간의 창 목록과 배치를 현재 상태로 교체합니다. `이름 변경`은 작업공간 이름만 바꾸며, `삭제`는 저장된 작업공간만 지우고 열려 있는 창은 변경하지 않습니다.

나머지 앱 설정과 앱 호환성 정책은 이 Mac의 TWMA 환경설정에 별도로 저장하며, 로그인 실행 상태는 macOS 로그인 항목이 관리합니다. 저장·복원 결과로 표시하는 호환 상태는 백업하지 않습니다.

Application Support 폴더는 현재 사용자만 접근할 수 있는 `700`, 프로필 파일은 `600` 권한으로 저장합니다.

## 로컬 진단 로그

앱은 드래그 스냅과 복원 실패를 확인하기 위한 기술 로그를 `~/Library/Logs/TWMA.log`에 저장합니다. 로그에는 창 제목, 사용자가 붙인 작업공간 이름과 URL을 기록하지 않습니다. 실행된 앱을 정확한 빌드와 연결하기 위해 앱 버전·빌드 번호·bundle identifier·소스 commit ID와 clean/dirty 상태를 기록합니다. 복원과 성능을 진단할 때 무작위 복원 세션 ID, 실행 계기, 모니터·성공·실패 창 수, 앱의 bundle identifier, 후보 창 수와 처리 시간이 기록될 수 있습니다.

로그는 512KB를 넘으면 `TWMA.previous.log` 한 개로 교체하며 계속 누적하지 않습니다. 두 로그 파일 모두 현재 사용자만 읽을 수 있는 `600` 권한을 사용합니다.

## 수집하거나 전송하지 않는 정보

- 계정, 비밀번호, 단축키 지정 화면에서 고른 조합 외의 키 입력, 화면 이미지
- 창 안의 문서 내용
- 분석·광고·추적 데이터
- 클라우드 또는 제3자 서버로 보내는 데이터

앱은 창을 읽고 이동하기 위해 macOS 손쉬운 사용 권한을 사용합니다. 활성 데스크탑에서 화면에 보이는 일반 앱 창만 저장 대상으로 삼습니다.

현재 베타에는 업데이트 피드와 인앱 업데이트 설정이 없습니다. 앱은 버전 확인·다운로드를 위한 네트워크 요청을 보내지 않으며, 실제 업데이트 서비스를 연결하기 전 요청 주소와 전송되는 기술 정보를 이 문서에 반영합니다.

사용자가 앱 안의 버그 제보·지원 링크를 누르면 macOS 기본 브라우저가 구성된 외부 HTTPS 페이지를 엽니다. 앱은 해당 페이지에 창·작업공간 데이터를 첨부하거나 자동 전송하지 않습니다.

## 사용자가 내보내는 백업

`앱 및 데이터`의 `백업 내보내기`에서 만든 JSON 파일에는 다음 정보가 포함됩니다.

- 저장된 작업공간의 이름·변경 시각·선택적 복원 단축키와 앱·창·모니터·레이아웃·배치 영역 정보
- 현재 앱 표시 언어, 미리보기 스타일, 드래그 배치, Dock 공간 확보, 창 사이 간격, 자동 복원 설정과 이전 버전 호환용 업데이트 확인 선택 값
- 기본·모니터별 레이아웃, 메뉴 즐겨찾기와 전역 1~4번 슬롯 단축키, 자동 복원 대상
- 사용자가 지정한 앱의 이름·bundle identifier와 `자동`·`위치만 복원`·`창 관리에서 제외` 모드
- 백업 생성 시각, 앱 버전과 백업 형식 버전

macOS 손쉬운 사용 권한, 로그인 항목 상태와 승인, 진단 로그, 첫 실행 권한 안내 확인 여부, 설정창 위치와 선택한 메뉴는 백업에 포함하지 않습니다.

백업 파일은 사용자가 선택한 위치에만 저장하며 앱이 자동으로 업로드하거나 동기화하지 않습니다. 파일 권한을 지원하는 저장소에서는 현재 사용자만 읽고 쓸 수 있는 `600` 권한을 적용합니다. 가져오는 프로필·백업 파일이 8 MiB를 넘으면 현재 데이터를 바꾸지 않고 거부합니다.

저장된 작업공간은 `작업공간` 화면의 `삭제`에서 개별 삭제할 수 있습니다. 전체 작업공간 데이터를 지우려면 앱을 종료한 뒤 `profiles-v1.json`을 삭제할 수 있습니다. 이 파일을 삭제해도 별도로 저장된 앱 설정, 앱 호환성 정책과 진단 로그는 삭제되지 않습니다.

공개 운영 표기는 `4celain`이며, 지원과 버그 제보는 [TWMA 공개 배포·지원 저장소](https://github.com/4celain/twma-releases/issues/new?template=bug_report.yml)에서 받습니다.
